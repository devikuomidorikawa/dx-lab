## 課題

月次の予算執行状況レポートを作成するために、経理担当者が財務会計システムからデータを出力し、スプレッドシートで集計し、前月比や執行率を計算し、Word でレポートにまとめていた。完成したレポートはメールで関係者に送付していたが、送付先の漏れや送信の遅れが発生することもあった。毎月同じ作業の繰り返しで、1レポートあたり数時間かかっていた。

## この記事が役立つケース

**導入が効果的な場合:**
- 月次の予算レポートを毎月作成しており、定型作業に数時間かかっている
- レポートの配布先が多く、共有の手間や漏れが発生している
- 異常値や予算超過の兆候に気づくのが遅れることがある

**現状のままで十分な場合:**
- レポートの作成頻度が四半期に1回程度で、負担が小さい
- 財務会計システムにレポート出力機能があり、そのまま配布できている
- レポートの配布先が2〜3名程度で、メール送付で十分に管理できている

## 解決策

財務会計システムから出力した CSV ファイルを **Google ドライブ** の指定フォルダに入れるだけで、あとはすべて自動で処理される仕組みを構築した。CSV のデータは **Google スプレッドシート** に自動取り込みされ、あらかじめ設定した集計処理が実行される。その結果を **Gemini** が分析してレポートを生成し、**Google ドキュメント** として保存する。完成したレポートは関係者が閲覧できる共有フォルダに自動コピーされ、**Google Chat** で通知が届く。**Google Workspace Studio の Flows** でこの一連の処理を自動化する。

## 仕組みの全体像

```
【経理担当者の操作】
  財務会計システムから CSV を出力
    → Google ドライブの「CSV 投入」フォルダに入れる
      （ここで操作は終了）

【自動処理（Flows）】
  CSV 投入を検知
    → スプレッドシートにデータを自動取り込み
      → あらかじめ設定した集計処理を実行
        → Gemini がデータを分析しレポートを生成
          → Google ドキュメントとして保存
            → 共有フォルダに自動コピー
              → Google Chat で関係者に通知
```

## 構築手順

### 1. フォルダ構成の準備

Google ドライブに以下のフォルダを作成する。

| フォルダ名 | 用途 | 権限設定 |
|-----------|------|---------|
| 予算管理_CSV投入 | 経理担当者が CSV を入れるフォルダ | 経理担当者のみ編集可 |
| 予算管理_レポート | 完成レポートの保存先 | 関係者（管理職・経理部門）が閲覧可 |

「予算管理_レポート」フォルダの共有設定で、レポートを確認する必要がある人（経理部門、管理職、学部長など）に閲覧権限を付与しておく。このフォルダにレポートがコピーされるだけで、自動的に全員がアクセスできる。

### 2. レポートの最終形を手動で一度作る

自動化の前に、**まず自分自身で理想のレポートを1回手作業で作成する。** これが自動処理のゴールになる。

**手順:**

1. 財務会計システムから CSV を出力する
2. スプレッドシートに CSV を取り込む
3. 自分で集計・加工し、最終的に報告に使いたい形に仕上げる

この時点では数式や自動化は気にせず、「報告書として見せたい最終形」を作ることだけに集中する。たとえば以下のような形になる。

| 科目名 | 予算額 | 当月執行額 | 累計執行額 | 執行率 | 残額 | 前月比 | ステータス |
|--------|--------|----------|----------|--------|------|--------|----------|
| 人件費 | 50,000,000 | 4,200,000 | 8,350,000 | 16.7% | 41,650,000 | +1.2% | 正常 |
| 旅費 | 3,000,000 | 280,000 | 430,000 | 14.3% | 2,570,000 | +86.7% | 要注意 |

### 3. Gemini に集計ルールを分析させる

手動で作成した最終形と元の CSV データの両方を Gemini に読み込ませ、「どのような処理をしたか」を言語化してもらう。

**手順:**

1. [gemini.google.com](https://gemini.google.com/) を開く
2. 元の CSV ファイルと、手動で作成した最終形のスプレッドシートを添付する
3. 以下のプロンプトを入力する

**プロンプト:**

```
添付した2つのファイルを比較してください。

1つ目は財務会計システムから出力した元データ（CSV）です。
2つ目は元データをもとに手動で作成した集計レポートの最終形です。

元データから最終形を作るために行った処理を、
以下の観点で漏れなく整理してください。

- 追加された列とその計算方法（数式レベルで記述）
- データの並べ替えやフィルタリング
- 書式の変更（パーセント表示、桁区切りなど）
- 条件に基づく分類やステータス付与のルール
- 元データにない情報がある場合、その導出方法

Google スプレッドシートの数式として実装できる形式で出力してください。
```

Gemini が出力する分析結果の例:

```
【追加列と計算方法】
- 執行率: =累計執行額/予算額（パーセント表示）
- 残額: =予算額-累計執行額
- 前月比: =(当月執行額-前月執行額)/前月執行額（パーセント表示）
- ステータス: =IF(執行率>0.9,"警告",IF(執行率>0.8,"要注意","正常"))

【データの加工】
- 科目コード列は最終形では削除されている
- 金額は3桁区切りで表示
- 執行率の降順で並べ替え
```

この分析結果が自分の意図と合っているか確認する。漏れや間違いがあれば修正を指示する。**確認が済んだら、この内容がそのまま自動集計の設計書になる。**

### 4. 集計用スプレッドシートの作成

Gemini が分析した集計ルールをもとに、スプレッドシートを構成する。

#### CSV 取り込みシート

財務会計システムから出力される CSV の形式に合わせたシート。Flows が CSV の内容をこのシートに書き込む。財務会計システムによって CSV の列構成は異なるため、実際のシステムに合わせて調整する。

#### 集計シート（数式で自動計算）

CSV 取り込みシートのデータを参照し、ステップ3で Gemini が分析した数式を設定する。CSV 取り込みシートのデータが更新されると、集計シートの値も自動的に再計算される。

前月のデータとの比較のため、過去の CSV 取り込みデータを別シートに月別で蓄積しておく。

### 5. Gemini によるレポート生成

Flows のステップで Gemini にスプレッドシートの集計結果を渡し、レポートを生成する。

**プロンプト:**

```
以下の予算執行データから月次レポートを作成してください。

【対象期間】{{年月}}

【集計データ】
{{スプレッドシートの集計シートのデータ}}

【レポートの構成】
1. 概要（3行程度で全体の状況を要約）
2. 科目別の執行状況（表形式: 科目名、予算額、累計執行額、執行率、残額）
3. 前月比で大きな変動がある科目の分析（変動率20%以上）
4. 執行率が高い（80%以上）または低い（10%未満）科目への注意喚起
5. 年度末までの見通し

簡潔かつ正確に記載してください。金額は3桁区切りで表記してください。
```

### 6. レポートの自動保存と共有フォルダへのコピー

Flows の後続ステップで以下を実行する。

1. Gemini が生成したレポートを Google ドキュメントとして作成
2. ファイル名を「予算執行状況レポート_YYYY年MM月」の形式で自動命名
3. 「予算管理_レポート」フォルダにコピー

共有フォルダにはあらかじめ関係者の閲覧権限が設定されているため、コピーされた時点で全員がアクセスできる。個別に権限を付与する必要はない。

### 7. Google Chat での通知

レポートの生成が完了したら、Google Chat のスペース（チャットルーム）に自動通知する。

**通知メッセージの例:**

```
📊 予算執行状況レポート（2026年2月分）が作成されました。

レポートはこちら: {{レポートのURL}}

【概要】
・全体の執行率: 68.5%（前月比 +3.2pt）
・要注意科目: 旅費（執行率 85.3%）
・残り予算が少ない科目: 消耗品費（残額 210,000円）
```

**Google Chat スペースの設定:**
1. 予算管理用の Google Chat スペースを作成する
2. レポートを確認する関係者をメンバーに追加する
3. Flows から Webhook URL 経由でメッセージを送信する

**Webhook URL の取得手順:**
1. Google Chat スペースを開く
2. スペース名の横にある「▼」→「アプリと統合」→「Webhook を管理」
3. 「Webhook を追加」でURLを取得
4. この URL を Flows の通知ステップに設定する

### 8. Flows の全体設定

Google Workspace Studio の Flows で、以下のステップを一つのフローとして構成する。

| ステップ | 処理内容 |
|---------|---------|
| トリガー | 「予算管理_CSV投入」フォルダにファイルが追加されたら実行 |
| ステップ 1 | CSV ファイルの内容を読み取る |
| ステップ 2 | 集計用スプレッドシートの CSV 取り込みシートにデータを書き込む |
| ステップ 3 | 集計シートの計算完了を待つ（数秒） |
| ステップ 4 | 集計シートのデータを取得し、Gemini にプロンプトとともに渡す |
| ステップ 5 | Gemini の出力を Google ドキュメントとして作成・命名 |
| ステップ 6 | ドキュメントを「予算管理_レポート」フォルダにコピー |
| ステップ 7 | Google Chat の Webhook にレポート URL と概要を送信 |

## 運用フロー

### 経理担当者の操作（毎月）

1. 財務会計システムから予算執行データの CSV を出力する
2. Google ドライブの「予算管理_CSV投入」フォルダに CSV を入れる
3. **（ここで操作は終了。あとはすべて自動）**
4. Google Chat に通知が届いたら、レポートの内容を確認する
5. 数値に問題がなければ、そのまま関係者が閲覧できる状態になっている

### 関係者（管理職・学部長など）の操作

1. Google Chat で通知を受け取る
2. リンクからレポートを確認する
3. 過去のレポートは「予算管理_レポート」フォルダにすべて蓄積されている

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| レポート作成時間 | 3〜4 時間 | 確認のみ（15 分） |
| 配布作業 | メールで個別送付 | 共有フォルダ＋Chat 通知で自動 |
| 配布漏れ | 送付先の追加忘れが発生 | フォルダ権限で一括管理 |
| 異常値の検出 | 目視で気づく | Gemini が自動で指摘 |
| 過去レポートの参照 | メールを検索 | フォルダに時系列で蓄積 |

## 応用例

- **四半期レポート**: 3か月分の CSV をまとめて投入し、四半期ごとの傾向分析を含むレポートを生成する
- **部門別レポート**: CSV を部門別に分割して投入し、部門ごとのレポートを自動生成する。各部門の共有フォルダに振り分けることも可能
- **科研費の執行管理**: 科研費など外部資金の執行状況を同じ仕組みで管理する。執行期限が近い場合の警告も Gemini に含めさせる

## 注意点

- 財務会計システムの CSV 形式が変更された場合は、スプレッドシートの取り込み設定とFlows の処理を修正する必要がある
- Gemini が生成した数値は必ず元データと突き合わせて確認する。特に金額の桁や合計値の整合性に注意する
- レポートの共有フォルダの権限設定は、人事異動に合わせて定期的に見直す
- CSV ファイル名に日本語や特殊文字が含まれると処理に失敗する場合がある。「budget_202602.csv」のように英数字で命名するルールを決めておく
