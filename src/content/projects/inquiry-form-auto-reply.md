## 課題

大学のホームページに掲載している問い合わせ先がメールアドレスや電話番号のみで、受験生・保護者・学外からの問い合わせが各部署にバラバラに届く。対応が属人的で、同じ質問に何度も一から回答を作成している。また、対応漏れや返信の遅れが発生しやすく、問い合わせがどこで止まっているかも把握しにくい。

## この記事が役立つケース

**導入が効果的な場合:**
- 問い合わせの件数が多く（目安: 月20件以上）、回答作成に時間がかかっている
- 同じような質問が繰り返し届いており、回答内容がほぼ定型化できる
- 対応漏れや返信遅れが発生しており、管理の仕組みが必要

**現状のままで十分な場合:**
- 問い合わせが月数件程度で、担当者が余裕を持って対応できている
- 問い合わせ内容が専門的で、AI では回答案を作りにくい

## 解決策

大学全体の問い合わせ窓口を **Google フォーム** で作成し、ホームページに公開する。問い合わせが届くと **Google スプレッドシート** に自動記録され、**Gemini** が回答案を生成する。回答案は問い合わせ内容とともに **Google Chat** のスペースに自動通知される。担当者は通知を見て回答案を確認・修正し、問い合わせ者に返信する。一連の自動処理は **Google Workspace Studio の Flows** で構築する。

## 仕組みの全体像

```
【問い合わせ者】
  大学ホームページから問い合わせフォームにアクセス
    → 必要事項を入力して送信

【自動処理（Flows）】
  フォーム送信を検知
    → スプレッドシートに自動記録
      → Gemini が問い合わせ内容を分析し回答案を生成
        → Google Chat のスペースに通知
          （問い合わせ内容 + 回答案 + スプレッドシートへのリンク）

【担当者】
  Google Chat で通知を受け取る
    → 回答案を確認・修正
      → 問い合わせ者にメールで返信
        → スプレッドシートの対応状況を「対応済み」に更新
```

## 構築手順

### 1. 問い合わせフォームの作成

Google フォームで大学全体の問い合わせフォームを作成する。

**フォームの項目:**

| 項目 | 入力形式 | 必須 | 備考 |
|------|---------|------|------|
| お名前 | 短文テキスト | 必須 | |
| メールアドレス | 短文テキスト | 必須 | メールアドレスの検証を有効にする |
| お立場 | プルダウン | 必須 | 受験生 / 在学生 / 保護者 / 卒業生 / 企業・団体 / その他 |
| 問い合わせカテゴリ | プルダウン | 必須 | 入試 / 学生生活 / 授業・履修 / 証明書 / 施設利用 / その他 |
| 問い合わせ内容 | 長文テキスト | 必須 | |

**フォーム設定:**
- 「回答のコピーを回答者に送信」を有効にする（問い合わせ者に控えが届く）
- 送信後のメッセージ: 「お問い合わせありがとうございます。内容を確認のうえ、3営業日以内にメールで回答いたします。」

### 2. フォームをホームページに公開する

作成したフォームの URL を大学ホームページの問い合わせページに掲載する。

**掲載方法:**
- リンクとして掲載する場合: フォームの URL をそのまま掲載
- ページ内に埋め込む場合: フォームの「送信」→「< >」（HTMLを埋め込む）から埋め込みコードを取得し、ホームページの HTML に貼り付ける

### 3. 回答案生成の参考情報を準備する

Gemini が的確な回答案を生成するために、よくある質問と回答（FAQ）や各部署の連絡先などの参考情報を Google ドキュメントにまとめておく。

**参考情報ドキュメントに含める内容:**

```
【入試関連】
Q: 出願方法を教えてください。
A: 出願は本学ホームページの入試情報ページから行えます。
   出願期間は○月○日〜○月○日です。
   詳細は入試要項をご確認ください。（URL: ...）

Q: オープンキャンパスの日程を教えてください。
A: 今年度のオープンキャンパスは○月○日と○月○日に開催予定です。
   詳細・申し込みはこちらをご覧ください。（URL: ...）

【証明書関連】
Q: 卒業証明書の発行方法を教えてください。
A: 卒業証明書は以下の方法で申請できます。
   窓口: 教務課（本館1階）受付時間 9:00-17:00
   郵送: 所定の申請書と手数料（1通300円分の定額小為替）を...

【各部署の連絡先】
- 入試課: nyushi@example.ac.jp（内線1234）
- 教務課: kyoumu@example.ac.jp（内線2345）
- 学生課: gakusei@example.ac.jp（内線3456）
- 総務課: soumu@example.ac.jp（内線4567）
```

既存の FAQ ページや大学案内から内容を転記すればよい。このドキュメントは Flows のプロンプト内で参照する。新しい質問パターンが増えたら随時追記する。

### 4. 管理用スプレッドシートの設定

フォームの回答が記録されるスプレッドシートに、管理用の列を追加する。

**シートの構成:**

| タイムスタンプ | お名前 | メールアドレス | お立場 | カテゴリ | 問い合わせ内容 | 回答案 | 対応状況 | 対応者 | 対応日 |
|--------------|--------|-------------|--------|---------|-------------|--------|---------|--------|--------|
| （自動） | （自動） | （自動） | （自動） | （自動） | （自動） | （Flows が記入） | 未対応 | | |

- 「タイムスタンプ」から「問い合わせ内容」まではフォーム送信時に自動記録
- 「回答案」は Flows が Gemini の出力を記入
- 「対応状況」「対応者」「対応日」は担当者が手動で更新

「対応状況」列にプルダウン（未対応 / 対応中 / 対応済み）を設定しておくと、フィルタで未対応の問い合わせをすぐに絞り込める。

### 5. Google Chat スペースの設定

問い合わせ通知を受け取る Google Chat スペースを作成する。

**手順:**
1. Google Chat で問い合わせ対応用のスペースを作成する（例: 「問い合わせフォーム対応」）
2. 対応する可能性がある担当者をメンバーに追加する
3. スペースの Webhook URL を取得する

**Webhook URL の取得手順:**
1. スペース名の横にある「▼」→「アプリと統合」→「Webhook を管理」
2. 「Webhook を追加」で名前を入力（例: 「問い合わせ通知」）し、URL を取得
3. この URL を Flows の通知ステップに設定する

### 6. Flows で自動処理を構築する

Google Workspace Studio の Flows で、フォーム送信から Chat 通知までの一連の処理を自動化する。

| ステップ | 処理内容 |
|---------|---------|
| トリガー | Google フォームに回答が送信されたら実行 |
| ステップ 1 | フォームの回答データ（名前、メールアドレス、立場、カテゴリ、内容）を取得 |
| ステップ 2 | Gemini に問い合わせ内容と参考情報を渡し、回答案を生成 |
| ステップ 3 | スプレッドシートの「回答案」列に Gemini の出力を記入 |
| ステップ 4 | Google Chat の Webhook に通知メッセージを送信 |

#### Gemini へのプロンプト（ステップ2）

```
あなたは大学の問い合わせ窓口の担当者です。
以下の問い合わせに対する回答案を作成してください。

【問い合わせ情報】
- お名前: {{名前}}
- お立場: {{立場}}
- カテゴリ: {{カテゴリ}}
- 問い合わせ内容: {{内容}}

【参考情報】
{{参考情報ドキュメントの内容}}

【回答作成のルール】
- 宛名は「{{名前}} 様」で始める
- 丁寧かつ簡潔な敬語で回答する
- 具体的な手続き方法、窓口、連絡先を含める
- 参考情報に該当する内容がない場合は、
  「担当部署に確認のうえ改めて回答いたします」として
  関連しそうな部署名を提案する
- URL やリンクを案内する場合は具体的に記載する
- 回答の末尾に「本メールの内容にご不明な点がございましたら、
  お気軽にお問い合わせください。」を添える
```

#### Chat 通知メッセージ（ステップ4）

```
📩 新しい問い合わせがありました

【問い合わせ者】{{名前}} 様（{{立場}}）
【カテゴリ】{{カテゴリ}}
【内容】
{{問い合わせ内容}}

【AI 回答案】
{{Gemini の回答案}}

⚠️ 回答案は AI が生成したものです。
内容を確認・修正のうえ、返信してください。

📋 管理シート: {{スプレッドシートの URL}}
```

### 7. 担当者の対応フロー

1. Google Chat で通知を受け取る
2. 回答案を確認する
3. 必要に応じて回答案を修正する（事実関係の確認、URL の追加など）
4. 問い合わせ者のメールアドレスに返信する
5. スプレッドシートの「対応状況」を「対応済み」に、「対応者」と「対応日」を記入する

回答案がそのまま使える場合は、コピーしてメールに貼り付けるだけで返信できる。

## 対応漏れの防止

### 未対応の問い合わせを定期的に通知する

Flows で毎朝、スプレッドシートの「対応状況」が「未対応」のまま一定期間経過した問い合わせを Google Chat に再通知する。

**通知メッセージの例:**

```
⏰ 未対応の問い合わせがあります

- {{名前}} 様（{{カテゴリ}}）- 受信から2日経過
- {{名前}} 様（{{カテゴリ}}）- 受信から1日経過

📋 管理シート: {{スプレッドシートの URL}}
```

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| 問い合わせの受付 | メール・電話がバラバラに届く | フォームに一元化 |
| 回答作成時間 | 1件あたり 10〜20 分 | AI 回答案の確認・修正で 2〜5 分 |
| 対応漏れ | 気づかないことがある | Chat 通知＋未対応リマインダーで防止 |
| 対応状況の把握 | 各担当者に聞かないとわからない | スプレッドシートで一元管理 |
| よくある質問への対応 | 毎回同じ内容を一から作成 | AI が参考情報をもとに自動生成 |

## 応用例

- **部署別のフォーム**: 入試課、教務課など部署ごとにフォームを分け、それぞれの Chat スペースに通知する。参考情報ドキュメントも部署別に用意すれば、より専門的な回答案を生成できる
- **多言語対応**: 問い合わせフォームに言語選択（日本語 / English）を追加し、英語の問い合わせには英語で回答案を生成するようプロンプトに条件を追加する
- **FAQ の自動更新**: スプレッドシートに蓄積された問い合わせと回答をもとに、定期的に参考情報ドキュメントや大学ホームページの FAQ を更新する

## 注意点

- AI が生成した回答案は必ず人間が確認してから返信する。特に入試日程、学費、手続き期限などの事実情報は、最新の情報と一致しているか確認する
- 参考情報ドキュメントは定期的に更新する。古い情報が残っていると、AI が誤った回答案を生成する原因になる
- 問い合わせフォームで収集する個人情報（氏名、メールアドレス）の取り扱いについて、大学のプライバシーポリシーに準拠する。フォームの冒頭に個人情報の利用目的を明記する
- フォームの送信後メッセージに回答までの目安日数を記載し、問い合わせ者の不安を軽減する
