## 課題

出張旅費の精算処理では、申請内容が学内の旅費規程に合致しているかを経理担当者が1件ずつ確認していた。日当の金額、宿泊費の上限、交通手段の妥当性など、チェック項目が多く、繁忙期には処理が滞ることがあった。

## 解決策

**Google スプレッドシート** に旅費規程のルールを整理し、**Gemini** を使って申請内容との照合を自動化した。

## フローの全体像

```
旅費精算申請（スプレッドシートに入力）
  → Gemini が申請内容と旅費規程を照合
    → 問題がある項目にフラグを立てる
      → 担当者はフラグのある申請のみ重点確認
```

## 構築手順

### 1. 旅費規程のルール整理

旅費規程の内容をスプレッドシートのルールシートに構造化する。

| 項目 | 条件 | 上限額 |
|------|------|--------|
| 日当（国内） | 日帰り | 2,000円 |
| 日当（国内） | 宿泊あり | 3,000円 |
| 宿泊費（国内） | 一般地域 | 10,000円 |
| 宿泊費（国内） | 特定地域（東京・大阪等） | 13,000円 |
| 交通費 | グリーン車 | 役職者のみ |

### 2. 申請データの入力シート

旅費精算申請を以下の形式でスプレッドシートに入力する。

| 申請者 | 出張先 | 日程 | 交通手段 | 交通費 | 宿泊費 | 日当 | 備考 |
|--------|--------|------|----------|--------|--------|------|------|

### 3. Gemini による自動チェック

Google スプレッドシートの Gemini 機能、または Google Apps Script から Gemini API を呼び出し、各申請をチェックする。

**プロンプト例:**

```
以下の出張旅費申請を旅費規程に照らしてチェックしてください。

【旅費規程】
{{ルールシートの内容}}

【申請内容】
{{申請データ}}

【チェック項目】
1. 日当の金額は規程の上限以内か
2. 宿泊費は地域区分に応じた上限以内か
3. 交通手段は規程に合致しているか
4. 日程と日当の日数は整合しているか

【出力フォーマット】
- 問題なし / 要確認
- 要確認の場合: 該当項目と理由を記載
```

### 4. 結果の反映

チェック結果をスプレッドシートの「チェック結果」列に自動入力する。「要確認」の行は背景色を変えて視覚的にわかりやすくする。

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| 1件あたりの確認時間 | 5〜10 分 | 1〜2 分（要確認分のみ） |
| 規程違反の見落とし | 月に数件 | ほぼゼロ |
| 繁忙期の残業 | 発生 | 大幅に減少 |

## 注意点

- Gemini のチェック結果は補助的なものとし、最終承認は担当者が行う
- 旅費規程が改定された場合は、ルールシートを速やかに更新する
- 特殊なケース（学会招待による旅費免除など）は個別に判断が必要
