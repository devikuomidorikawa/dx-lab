## 課題

稟議書や起案文書の作成は、過去の類似文書を探して参考にしながら書くことが多いが、適切な文書を見つけるのに時間がかかる。また、作成者によって文体や書式にばらつきがあり、差し戻しが発生することもあった。

## この記事が役立つケース

**導入が効果的な場合:**
- 稟議書や起案文書の作成頻度が高く、作成に時間がかかっている
- 書式や文体のばらつきで差し戻しが発生している
- 過去の類似文書を探すのに時間がかかる

**現状のままで十分な場合:**
- 起案の頻度が低く、テンプレートをコピーして修正する程度で済んでいる
- 決まった書式のテンプレートが整備されており、作成に迷うことがない

## 解決策

過去の起案文書を **NotebookLM** に読み込ませ、**Gemini** で新しい起案のドラフトを自動生成する仕組みを構築した。用件を簡潔に伝えるだけで、過去の文書の文体・書式に合わせたドラフトが生成される。

## 構築手順

### 1. 過去文書の収集

過去に決裁された起案文書（稟議書、伺い、通知文など）を Google ドライブのフォルダに整理する。

```
起案文書アーカイブ/
  ├── 物品購入/
  ├── 業務委託/
  ├── 出張/
  ├── イベント開催/
  └── 規程改正/
```

### 2. NotebookLM に読み込み

カテゴリごとに NotebookLM のノートブックを作成し、過去文書をソースとしてアップロードする。

### 3. ドラフト生成

用件を入力するだけで、過去文書のスタイルに合わせたドラフトが生成される。

**入力例:**

```
以下の内容で物品購入の稟議書ドラフトを作成してください。
過去の類似文書の書式・文体に合わせてください。

【概要】
- 購入物品: ノートPC 10台
- 用途: 新規採用職員用
- 予算: 約150万円
- 希望納期: 2025年4月1日まで
- 購入先: 3社から見積もり取得済み
```

**出力例:**

```
件名: ノートPC購入について（伺い）

1. 趣旨
   令和7年4月採用予定の新規職員に配備するノートPCを
   購入したく、下記のとおり伺います。

2. 購入内容
   品名: ノートPC
   数量: 10台
   金額: 1,500,000円（税込）

3. 購入理由
   ...

4. 見積もり
   別紙のとおり3社から見積もりを取得し、
   最低価格の○○社を選定しました。

5. 予算措置
   ○○費（科目コード: ○○）から支出予定
```

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| ドラフト作成時間 | 30 分〜1 時間 | 5〜10 分 |
| 過去文書の検索時間 | 10〜20 分 | 不要（AI が参照） |
| 書式・文体のばらつき | 作成者により差あり | 過去文書に準拠し統一 |
| 差し戻し率 | 月に数件 | 大幅に減少 |

## 応用例

- **通知文の作成**: 学内通知、依頼文、案内文のドラフト生成
- **報告書の作成**: 事業報告書、活動報告書のテンプレート生成
- **会議の付議資料**: 委員会付議のための資料ドラフト作成

## 注意点

- 生成されたドラフトは必ず内容を確認し、事実関係や金額の正確性を検証する
- 機密性の高い内容（人事案件など）は AI に入力しない運用ルールを設ける
- 過去文書のアーカイブは定期的に更新し、古い書式の文書は除外する
