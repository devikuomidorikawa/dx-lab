## 課題

毎朝大量の未読メールを1件ずつ確認し、対応の優先度を判断する作業に時間がかかっていた。重要なメールの見落としや、対応の遅れが発生することもあった。

## 解決策

**Google Workspace Studio の Flows** を使い、未読メールを AI が自動で分類・要約し、毎朝1通のサマリーメールとして配信するフローを構築した。

## フローの全体像

```
毎朝 8:00（スケジュールトリガー）
  → Gmail から未読の重要メールを取得
    → Gemini が内容を分析し、優先度別に分類
      → サマリーメールを自分宛に送信
```

## 構築手順

### 1. Flows の新規作成

Google Workspace Studio にアクセスし、テンプレート「Get a daily summary of unread emails」を選択する。このテンプレートをベースにカスタマイズする。

### 2. スケジュールトリガーの設定

- **実行タイミング**: 毎日（月〜金）
- **実行時間**: 午前 8:00
- **開始日**: 任意の日付

### 3. Gemini プロンプトのカスタマイズ

テンプレートのプロンプトを以下のように書き換え、日本語で優先度別に整理させる。

**プロンプト例:**

```
以下の未読メールを分析し、日本語で日次サマリーを作成してください。

【分類ルール】
1. 緊急対応（本日中に返信・対応が必要なもの）
2. 要確認（今週中に目を通すべきもの）
3. 参考情報（急ぎではないが把握しておくべきもの）

【出力フォーマット】
■ 緊急対応
  - 件名: ...
    差出人: ...
    要旨: （1〜2行で要約）
    推奨アクション: ...

■ 要確認
  - 件名: ...
    差出人: ...
    要旨: ...

■ 参考情報
  - 件名: ...
    要旨: ...

対象メールがない分類は「該当なし」と記載してください。
```

### 4. 通知メールの設定

- **宛先**: 自分のメールアドレス
- **件名**: `【日次サマリー】未読メール要約`
- **本文**: Gemini が生成したサマリー

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| 朝のメール確認時間 | 30〜45 分 | 5 分（サマリー確認のみ） |
| 重要メールの見落とし | 月に数件発生 | ほぼゼロ |
| 対応の優先判断 | 1件ずつ手動で判断 | AI が自動分類 |

## 注意点

- テンプレートの初期設定では「重要なメール」のみが対象となる。Gmail が自動判定した重要メール、またはユーザーが重要マークを付けたメールが抽出される
- サマリーの精度はメール本文の内容に依存するため、添付ファイルの中身までは要約されない
- 機密情報を含むメールがサマリーに含まれる可能性があるため、転送設定には注意が必要
