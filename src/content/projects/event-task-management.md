## 課題

入学式、卒業式、オープンキャンパスなどの学内行事は、準備タスクが多岐にわたり、複数の部署が関わる。タスクの進捗管理が属人化しており、抜け漏れや期限超過が発生することがあった。

## この記事が役立つケース

**導入が効果的な場合:**
- 行事の準備タスクが多岐にわたり、複数の部署が関わっている
- タスクの抜け漏れや期限超過が実際に発生したことがある
- 進捗確認のために個別に聞き回る手間がかかっている

**現状のままで十分な場合:**
- 行事の規模が小さく、担当者が1〜2名で管理できている
- 既にプロジェクト管理ツール（Asanaなど）を導入している

## 解決策

**Google スプレッドシート** で行事ごとのタスク管理表を作成し、**Google Workspace Studio の Flows** で期限のリマインダーや進捗の定期報告を自動化した。

## 構築手順

### 1. タスク管理テンプレートの作成

行事ごとにスプレッドシートのテンプレートを作成する。

| タスク | 担当部署 | 担当者 | 期限 | ステータス | 備考 |
|--------|---------|--------|------|-----------|------|
| 式次第の作成 | 総務課 | 田中 | 3/1 | 完了 | |
| 来賓リストの確認 | 総務課 | 鈴木 | 3/5 | 進行中 | |
| 会場設営の手配 | 施設課 | 佐藤 | 3/10 | 未着手 | |
| 案内状の送付 | 広報課 | 山田 | 2/28 | 完了 | |

### 2. Flows によるリマインダー

毎朝、期限が3日以内に迫っているタスクの担当者にリマインダーメールを自動送信する。

### 3. 週次進捗レポートの自動生成

毎週月曜日に、タスクの進捗状況をまとめたレポートを Gemini で生成し、関係者にメールで配信する。

**プロンプト例:**

```
以下のタスク管理表から、今週の進捗レポートを作成してください。

【レポート項目】
- 完了したタスク
- 進行中のタスク
- 期限超過のタスク（あれば強調）
- 今週中に期限を迎えるタスク
- 全体の進捗率（完了数 / 全タスク数）

簡潔にまとめてください。
```

### 4. 行事終了後の振り返り

行事終了後、タスク管理表の記録をもとに、Gemini で振り返りレポートを生成し、次回の改善に活かす。

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| タスクの抜け漏れ | 年に数件発生 | ほぼゼロ |
| 進捗確認 | 個別に聞き回る | スプレッドシートで一目瞭然 |
| リマインダー | 手動で連絡 | 自動メール |
| 過去の記録 | 残っていない | テンプレートとして蓄積 |

## 応用例

- **入試業務**: 出願受付から合格発表までのタスク管理
- **認証評価の準備**: 自己点検・評価に必要な書類収集のタスク管理
- **年度末業務**: 決算、人事異動、新年度準備のタスク管理

## 注意点

- タスクの粒度は細かすぎず、1タスク = 1アクションを目安にする
- 担当者が自分でステータスを更新するルールを徹底する
- 前回の行事のタスク管理表をテンプレートとして次回に流用し、毎回ゼロから作らない
