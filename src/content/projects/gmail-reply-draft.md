## 課題

大学事務では、教員・学生・外部機関など立場の異なる相手に日常的にメールを送信する。丁寧な文面を一から考えると1通あたり10〜15分かかることもあり、件数が多い日は大きな負担になる。また、Gemini で返信案を生成しても一般的な表現になりがちで、「自分が書いた感じ」にならず、結局大幅に書き直すことになりやすい。

## この記事が役立つケース

**導入が効果的な場合:**
- メールの送受信件数が多く、返信に時間がかかっている
- 文面を一から考えることが多く、テンプレートだけでは対応しきれない
- AI が生成する文面が自分のスタイルと合わず、毎回修正が多い

**現状のままで十分な場合:**
- メールの件数が少なく、作成に困っていない
- 定型文のテンプレートで大半のメールをカバーできている

## 解決策

**Gmail** に搭載された **Gemini** の機能を使い、受信メールへの返信案や新規メールのドラフトを生成する。さらに、過去の送信メールを **NotebookLM** に読み込ませて「自分のメールスタイルガイド」を作成すれば、自分らしい文面を Gemini に生成させることもできる。

## 前提条件

- Google Workspace Business Standard 以上、または Google One AI Premium プラン
- Gmail の Gemini 機能が管理者により有効化されていること

## 返信メールの作成

### 1. 返信案の自動提案

受信メールを開いて「返信」をクリックすると、メール本文の下に Gemini が提案する返信案がチップ（ボタン）として表示される。

**表示される返信案の例:**
- 「承知しました」
- 「ありがとうございます」
- 「確認して折り返します」

チップをクリックすると、その趣旨に沿った丁寧な返信文が自動生成される。内容を確認・修正して送信する。

### 2. 自分で指示して返信を作成する

提案されたチップが合わない場合は、自分で指示を出して返信を作成できる。

**手順:**

1. 受信メールを開き「返信」をクリック
2. 返信作成画面の下部にある **Gemini アイコン（ペンのマーク）** をクリック
3. 「このメールへの返信を手伝う」を選択
4. 作成したい返信の要旨を入力する

**入力例:**

```
3月15日の会議について、出席できる旨を伝えてください。
資料は事前に確認しておく旨も添えてください。
```

```
申請の件について、添付の書類に不備があるため
再提出をお願いする返信を作成してください。
丁寧な表現でお願いします。
```

```
この問い合わせに対して、担当部署は教務課であること、
教務課の連絡先（内線1234）を案内する返信を作成してください。
```

5. Gemini が返信文を生成する
6. 内容を確認し、必要に応じて修正して送信

### 3. 文体の調整

生成された返信文は、以下のオプションで調整できる。

| オプション | 効果 |
|-----------|------|
| フォーマルにする | より丁寧な敬語表現に調整 |
| カジュアルにする | 親しみのある表現に調整 |
| 短くする | 冗長な表現を削り簡潔にする |
| 詳しくする | 説明を補足して丁寧にする |

相手に応じて使い分ける。教員や外部機関向けには「フォーマル」、日常的にやり取りのある同僚には「カジュアル」が適している。

## 新規メールの作成

Gemini は返信だけでなく、新規メールの作成もサポートしている。

**手順:**

1. Gmail で「作成」をクリックし、新規メール作成画面を開く
2. 下部の **Gemini アイコン（ペンのマーク）** をクリック
3. 「メールの作成を手伝う」を選択
4. 作成したいメールの内容を指示する

**入力例（会議の案内）:**

```
教務委員会の委員宛に、次回会議の案内メールを作成してください。

日時: 3月20日（木）14:00〜15:30
場所: 本館3階 会議室A
議題: 来年度カリキュラム改定について
持ち物: 配布済みの改定案

出欠の回答を3月17日までにお願いする旨を含めてください。
```

**入力例（依頼メール）:**

```
総務課の鈴木さん宛に、会議室の予約を依頼するメールを作成してください。
4月10日の午後、20名程度が入れる部屋を希望します。
プロジェクターも使いたい旨を伝えてください。
```

**入力例（報告メール）:**

```
部長宛に、先日のオープンキャンパスの実施報告メールを作成してください。
来場者数は約300名、昨年比120%でした。
アンケート結果は概ね好評で、詳細は別途報告書にまとめる予定です。
```

5. Gemini がメール文面を生成する
6. 宛先・件名を入力し、内容を確認・修正して送信

## 活用のコツ

- **指示は具体的に** — 「お礼のメールを書いて」より「3月10日の研修会の講師を務めていただいたお礼のメールを書いて」の方が的確な文面が生成される
- **相手の立場を明示する** — 「教員宛に」「学生宛に」「外部の取引先宛に」と指示すると、適切な敬語レベルで作成される
- **含めたい情報を箇条書きで渡す** — 日時・場所・締切などの具体的な情報を箇条書きで指示すると、漏れなく文面に反映される
- **生成後に「フォーマル」「短く」などで微調整する** — 一発で完璧な文面にならなくても、調整オプションで仕上げられる

## 自分のメールスタイルを学習させる

Gmail の Gemini が生成する文面は一般的な表現になりがちで、自分の言い回しや構成のクセが反映されない。過去のメールから自分のスタイルを抽出し、それに沿った文面を生成させることで、修正の手間を大幅に減らせる。

### 1. 過去の送信メールを収集する

Gmail の「送信済み」から、自分らしさがよく出ている代表的なメールを10〜20通選ぶ。

**選ぶ際のポイント:**
- 相手の立場ごとに数通ずつ（教員宛、学生宛、外部機関宛、同僚宛）
- よく書く種類のメール（依頼、回答、案内、お礼、お詫びなど）
- 自分で「これはうまく書けた」と思うメール

Gmail で送信メールを開き、本文をコピーして Google ドキュメントに貼り付ける。1通ごとに区切りを入れると分析しやすい。

```
========== メール1（教員宛・依頼）==========
件名: ○○の件についてお願い
宛先: 教員

（本文を貼り付け）

========== メール2（学生宛・回答）==========
件名: お問い合わせへの回答
宛先: 学生

（本文を貼り付け）

========== メール3（外部機関宛・案内）==========
...
```

### 2. NotebookLM で自分のスタイルを分析する

1. [NotebookLM](https://notebooklm.google.com/) で新しいノートブックを作成
2. 収集したメールの Google ドキュメントをソースとして追加
3. 以下のプロンプトでスタイルを分析させる

**プロンプト:**

```
アップロードしたメール文面を分析し、
この人のメールの書き方の特徴を「メールスタイルガイド」として
まとめてください。

以下の観点で分析してください。

■ 全体の構成パターン
  - メールの冒頭はどう始めるか
  - 本題への入り方
  - 締めの定型表現
  - 署名の前の一言

■ 敬語のレベルと特徴
  - どの程度フォーマルか
  - よく使う敬語表現
  - 避けている表現

■ 特徴的な言い回し・口癖
  - よく使うフレーズ
  - 接続詞の使い方
  - クッション言葉の使い方

■ 相手別の書き分け
  - 教員宛と学生宛で変わる点
  - 外部向けと学内向けで変わる点

■ その他の特徴
  - 1文の長さの傾向
  - 箇条書きの使用頻度
  - 改行の入れ方

具体的なメール文面から実例を引用しながらまとめてください。
```

### 3. スタイルガイドを保存する

NotebookLM が生成した分析結果を確認し、自分の認識と合っているか確認する。合っていれば、Google ドキュメントにコピーして保存する。

**スタイルガイドの出力例:**

```
【○○さんのメールスタイルガイド】

■ 構成パターン
- 冒頭: 「○○先生」「○○さん」の後、「お疲れさまです。」で始める
- 本題: 用件がある場合「○○の件でご連絡いたしました。」で切り出す
- 補足: 詳細は箇条書きでまとめる傾向がある
- 締め: 「お忙しいところ恐れ入りますが、よろしくお願いいたします。」

■ 敬語の特徴
- 「いただけますでしょうか」より「いただけますか」を使う
- 「させていただく」の多用を避ける
- 教員には「先生」、職員には「さん」で統一

■ 特徴的な表現
- 依頼時: 「お手数ですが」「差し支えなければ」
- 確認時: 「念のため確認ですが」
- 返信不要の場合: 「特にご返信は不要です」
- 箇条書きの前: 「下記のとおりです。」

■ 相手別の違い
- 教員宛: より丁寧、1文を短くする
- 学生宛: 丁寧語ベース、具体的な手順を示す
- 同僚宛: 簡潔、「〜です」「〜します」で統一
```

内容に過不足があれば、手動で修正・追加する。このガイドが今後の「自分らしいメール」の基準になる。

### 4. スタイルガイドを使って返信案を作成する

日常のメール作成では、スタイルガイドを参照しながら Gemini に返信案を生成させる。

**方法 A: NotebookLM を使う（推奨）**

スタイルガイドをソースとして追加した NotebookLM のノートブックをそのまま使う。

```
以下のメールに対する返信を、
スタイルガイドに従って作成してください。

【受信メール】
（返信したいメールの本文を貼り付け）

【返信の要旨】
- 3月20日の会議に出席できることを伝える
- 資料は事前に確認しておく
- 当日の持ち物について質問する
```

NotebookLM はソースとしてスタイルガイドを保持しているため、毎回貼り付ける必要がない。

**方法 B: Gemini（gemini.google.com）を使う**

Gemini アプリにスタイルガイドを貼り付けて指示する。

```
以下のスタイルガイドに従って、メールの返信案を作成してください。

【スタイルガイド】
（ガイドの内容を貼り付け）

【受信メール】
（返信したいメールの本文を貼り付け）

【返信の要旨】
- 日程変更のお詫び
- 新しい候補日を3つ提示
```

**方法 C: Google ドキュメントの Gemini を使う**

Google ドキュメントにスタイルガイドを記載しておき、同じドキュメント内で Gemini サイドパネルに指示を出す。ドキュメント内のスタイルガイドを参照して文面を生成してくれる。

### 5. スタイルガイドを育てる

運用しながらスタイルガイドを改善する。

- Gemini が生成した文面で「自分ならこう書かない」と感じた表現があれば、ガイドの「避ける表現」に追加する
- 新しいパターンのメール（お詫び、異動の挨拶など）を書いたら、その文面もソースに追加する
- 半年〜1年に一度、ガイド全体を見直す

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| 返信メール作成時間 | 1通あたり 5〜15 分 | 1〜3 分（確認・修正のみ） |
| 新規メール作成時間 | 1通あたり 10〜20 分 | 2〜5 分 |
| AI 生成文の修正量 | 大幅に書き直し | スタイルガイドで微修正に |
| 敬語・表現への不安 | 特に新任者に多い | AI が適切な表現を提案 |
| 後任者への引き継ぎ | メールの書き方が伝わらない | ガイドを共有できる |

## 応用例

- **チーム共通のスタイルガイド**: 部署全体のメール文体を統一したい場合、複数人の良いメールをまとめて分析し、チーム共通ガイドを作成する
- **相手別のガイド**: 教員向け・外部向けなど、相手の立場ごとにガイドを分けて精度を高める
- **メール以外の文書**: 学内通知文や報告書など、メール以外の文書にもスタイルガイドを応用できる

## 注意点

- 生成された文面は必ず送信前に内容を確認する。事実関係（日時、金額、固有名詞）が正確かどうかは自分で検証する
- 収集するメールに個人情報や機密情報が含まれる場合は、該当部分を削除してからアップロードする
- スタイルガイドはあくまで「傾向」であり、すべてのメールに画一的に適用するものではない。状況に応じて柔軟に調整する
- NotebookLM の共有リンクにはスタイルガイドの内容が含まれるため、共有範囲に注意する
