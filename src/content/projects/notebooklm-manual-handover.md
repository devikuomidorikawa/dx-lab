## 課題

学内マニュアルの作成には時間がかかり、作成しても「読まれない」「読んでもわからない」という問題が発生しやすい。担当者が異動・退職する際の引き継ぎでは、マニュアルだけでは伝わらない暗黙知が多く、後任者が同じ質問を何度も前任者に問い合わせることになっていた。

## 解決策

**NotebookLM** を2つの役割で活用する。

1. **マニュアル作成の補助** — 既存の業務資料を読み込ませ、マニュアルのドラフトを生成
2. **質問対応 Bot** — 完成したマニュアルを読み込んだ NotebookLM のリンクをマニュアル内に掲載し、後任者がいつでも対話形式で質問できる環境を提供

これにより「マニュアルを読む → わからない → 人に聞く」のサイクルが「マニュアルを読む → わからない → Bot に聞く」に変わり、前任者への問い合わせが大幅に減る。

## 全体の流れ

```
【マニュアル作成フェーズ】
既存の業務資料（手順書・メモ・議事録など）
  → NotebookLM にアップロード
    → 対話しながらマニュアルのドラフトを生成
      → Google ドキュメントとして整形・完成

【引き継ぎフェーズ】
完成したマニュアル + 関連資料
  → 別の NotebookLM ノートブックにアップロード（質問 Bot 用）
    → 共有リンクをマニュアル内に記載
      → 後任者はマニュアルを読みつつ、不明点は Bot に質問
```

## 構築手順

### Phase 1: マニュアル作成

#### 1. 既存資料の収集

業務に関する資料をすべて集める。完璧に整理されていなくても問題ない。

- 既存の手順書・作業メモ
- 業務で使うスプレッドシートやテンプレート
- 過去の引き継ぎ資料
- 関連する学内規則
- 業務に関するメールのやり取り（要点のみ）

#### 2. NotebookLM に読み込ませる

1. NotebookLM で新しいノートブックを作成（例: 「○○業務マニュアル作成用」）
2. 収集した資料をソースとしてアップロード

#### 3. 対話しながらマニュアルを生成する

NotebookLM に質問・指示を出しながら、マニュアルの各セクションを作成する。

**業務の全体像を整理する:**

```
アップロードした資料をもとに、この業務の全体像を整理してください。
主な業務内容、年間スケジュール、関係する部署を一覧にしてください。
```

**手順書を生成する:**

```
○○申請の処理手順を、ステップごとに詳しく書き出してください。
各ステップで使うシステムやツール、注意点も含めてください。
```

**よくあるトラブルと対処法を抽出する:**

```
過去の資料やメモから、よくあるトラブルや例外的なケースを
抽出して、対処法とともにまとめてください。
```

**判断基準を明文化する:**

```
この業務で判断に迷いやすいポイントはどこですか？
資料から読み取れる判断基準やルールを整理してください。
```

#### 4. Google ドキュメントとして仕上げる

NotebookLM で生成した内容を Google ドキュメントにコピーし、構成を整える。

### Phase 2: 質問 Bot の構築

#### 1. 質問 Bot 用のノートブックを作成

新しい NotebookLM ノートブックを作成する（例: 「○○業務 質問 Bot」）。マニュアル作成用とは別のノートブックにする。

#### 2. 完成したマニュアルと関連資料をアップロード

以下をソースとして追加する。

- 完成したマニュアル（Google ドキュメント or PDF）
- 業務で使うテンプレートやフォーマット
- 関連する学内規則
- FAQ（あれば）

#### 3. 動作確認

後任者の立場で質問を投げかけ、適切な回答が返るか確認する。

```
○○申請の締め切りはいつですか？
```

```
この業務で使うシステムのログイン方法を教えてください。
```

```
イレギュラーな申請が来た場合、どう対応すればいいですか？
```

#### 4. マニュアルにリンクを埋め込む

マニュアルの冒頭または末尾に、質問 Bot へのリンクを記載する。

**記載例:**

```
【このマニュアルについて質問がある場合】
以下のリンクから、AI アシスタントに質問できます。
マニュアルの内容に基づいて回答します。
→ https://notebooklm.google.com/notebook/xxxxx
```

## 導入効果

| 項目 | 導入前 | 導入後 |
|------|--------|--------|
| マニュアル作成時間 | 数日〜数週間 | 数時間 |
| 引き継ぎ期間 | 2〜4 週間 | 1 週間程度 |
| 前任者への問い合わせ | 頻繁（日に数回） | 大幅に減少 |
| 暗黙知の伝達 | 口頭に依存 | 資料から AI が抽出 |
| マニュアルの理解度 | 読んでもわからない部分が残る | 不明点を Bot に即質問 |

## 応用例

- **新入社員研修**: 研修資料を NotebookLM に読み込ませ、新入社員がいつでも質問できる環境を用意
- **システム運用マニュアル**: 障害対応手順書を読み込ませ、障害発生時に即座に手順を確認
- **部署横断の共有知識**: 複数部署の業務資料をまとめて読み込ませ、組織全体のナレッジベースとして活用

## 注意点

- NotebookLM の共有リンクを使う場合、アクセス権限の設定に注意する。組織外に公開されないよう Google Workspace の管理設定を確認すること
- マニュアルの内容が更新された場合、質問 Bot 側のソースも差し替える必要がある
- Bot の回答はソース資料に基づくが、最終的な判断は上司や担当者に確認する運用ルールを設けておくとよい
- 機密情報を含む場合は、ノートブックの共有範囲を組織内に限定すること
